# 1. Теоретична частина (≈9 стор.)

> Вимоги до наповнення:
> - **≈2 стор.**: астрономія + математика + датасет.
> - **≈7 стор.**: суто вебдизайн (UI/UX) для вашого вебдодатку-візуалізації.
> - Опис технологій подати в теорії як «що це» + «де використано у проєкті».

## 1.1. Використані технології вебдодатку: призначення та роль у системі (≈2 стор.)

### 1.1.1. Backend (серверна частина)
- **Django (Python web framework)**: що це, MVC/MVT-архітектура; роль у проєкті (маршрутизація, шаблон `templates/index.html`, роздача `static/`, інтеграція з БД).
- **Django REST Framework (DRF)**: що це; роль у проєкті (формування REST API для режимів Explore/Random/Search/Compare/Kepler Lab).
- **SQLite**: що це; роль у проєкті (локальне сховище орбітальних елементів, швидкий старт без окремого сервера БД).

### 1.1.2. Frontend (клієнтська частина)
- **JavaScript ES Modules + importmap**: що це; роль у проєкті (підключення модулів у браузері без Node-білду, карта імпортів для `three`).
- **Three.js**: що це; роль у проєкті (3D-сцена, рендеринг планет, орбіт, поля малих тіл).
- **OrbitControls**: що це; роль (навігація камерою: обертання/масштаб/панорамування).
- **Postprocessing (EffectComposer + UnrealBloomPass)**: що це; роль (контрольований glow/bloom для підсилення контрасту й “космічної” атмосфери).
- **HTML + Django templates**: що це; роль (структура layout: sidebar + canvas + HUD/tooltip, importmap, точка входу `main.js`).
- **CSS (dark UI)**: що це; роль (візуальна система змінних, компоненти UI, сітка, стани hover/active).
- **i18n (UA/EN)**: що це; роль (словники, перемикання мови у UI та в підписах).
